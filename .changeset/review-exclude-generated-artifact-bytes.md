`totem review` now excludes generated-artifact BYTES from the LLM synthesis input and injects a summary instead of a silent drop (mmnto-ai/totem#2398). Previously the only exclusion mechanism was config `ignorePatterns` — opt-in, empty by default, and a silent drop: an excluded file simply vanished from the payload, losing the signal that it changed at all. Nothing classified generated artifacts by default, so a PR that regenerated lockfiles / `compiled-rules.json` / `dist/**` / `*.wasm` burned review-context tokens on bytes no reviewer should read.

A default generated-artifact classification (seeded globs — dependency lockfiles, Totem compiled artifacts, conventional build-output directories, compiled/minified/WASM outputs — plus any `.gitattributes` `linguist-generated` patterns the repo declares) now runs before the diff is assembled into the prompt. Each matched file's diff section is stripped from the review payload and replaced by a per-file summary line — path, change shape (added/deleted/regenerated), size delta (unified-diff insert/delete counts, or `binary`), and a semantic hash (sha256/12 of the excluded file's diff segment) — so the "this regenerated" signal survives without the bytes. The summary block is CLEARLY LABELLED as a Totem-generated summarization artifact (a distinct `<generated_artifacts_summary>` XML tag plus a preamble stating it is "not diff content"), the #2329 mislabel guard, so the model cannot mistake it for real diff content. Every excluded file is also loudly named on stderr — never a silent drop.

The classification is Git-native and escape-hatched: a repo un-marks a false positive with `.gitattributes` `linguist-generated=false` (or `-linguist-generated`) and adds its own generated globs with `<pattern> linguist-generated`. When nothing matches, the review payload is byte-identical to before (legacy path preserved). When every changed file is a generated artifact, the review takes a deterministic PASS after surfacing the summary. Aligns with the Prop 297 taxonomy — GENERATED spans take regeneration-diff treatment, never prose review.
