## Lesson — Proactively remove references to planned features from user

**Tags:** style, curated
**Pattern:** \b(coming soon|planned (feature|for)|future release|roadmap|under development|slated for|later development phases)\b
**Engine:** regex
**Scope:** **/\*.md, **/_.mdx, docs/\*\*/_, guides/**/\*
**Severity:\*\* warning

Proactively remove references to planned features from user guides to avoid misleading users. Documentation should reflect the current release state.
