## Lesson — Research spikes should minimize tooling overhead

**Tags:** workflow, architecture, spike

Research spikes exist to buy information cheaply. Do not build automated test harnesses, Zod schemas, or benchmark frameworks for exploratory experiments. The deliverable is a decision document, not code. Try the experiment directly, document what breaks, and write findings. Spike code is throwaway — writing tests for it is pure overhead. Fast failure beats pristine measurement.
