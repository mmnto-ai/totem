## Lesson — Target project-named packages for version badges

**Tags:** documentation, monorepo, dx
**Scope:** README.md

In monorepos with lockstep versioning, pointing the README badge to the project-named package (e.g., @mmnto/totem) instead of a sub-package (e.g., @mmnto/cli) clarifies the project-level version for users who might otherwise assume the version only applies to a specific tool.
