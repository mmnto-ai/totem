## Lesson — Never use git add -A or git add .

**Tags:** git, release, trap, worktree, ci

Never use `git add -A` or `git add .` when committing version bumps or release changesets. The working directory may contain agent worktree copies, runtime artifacts (ledger events, lock files), or cache files that should not be committed. Always stage specific files by name: `git add packages/*/package.json packages/*/CHANGELOG.md .changeset/`.
