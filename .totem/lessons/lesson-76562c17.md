## Lesson — The Git -- separator treats all subsequent arguments

**Tags:** architecture, curated
**Pattern:** \bgit\s+._\s--\s+._(\.\.\.?|[\^~])
**Engine:** regex
**Scope:** _.sh, _.bash, _.yml, _.yaml, Makefile, **/\*.ts, **/_.js, \*\*/_.py
**Severity:** error

Git revision specifiers (e.g., 'branch..HEAD', 'HEAD~') must appear before the '--' separator. Arguments after '--' are strictly interpreted as file paths.
