## Implementation Design — rev 4 (totem-claude; rev 1 2026-07-10, rev 2 after panel round 1, rev 3 after codex's rev-2 re-check, rev 4 2026-07-11 after the dual codex as-built round — 4 gates folded + operator G5 ruling + 14 as-built findings accepted)

### Scope

Build the Prop 302 **verdict artifact** (the single convergence point both lanes emit), wire the shipped panel-synthesis + post-check engines into `totem review` as an opt-in multi-lane fan, add fix-delta **round chaining** whose state machine lives entirely in the CLI (any agent gets identical loop capability — Tenet 16), and ship the **warm-lane distributed skill** (`review-loop`) as a thin driver: invoke → apply fixes → re-invoke until the CLI reports settled; no loop logic in the skill. Explicitly NOT in scope: the cold-lane SDK runner (reference-only, later slice; today's single-lane `totem review` remains the de-facto headless path); rule-corpus certification / aggregator work (stays gated, R4); durable disposition semantics (#2105 — nothing here implies it); pre-push hook-generator changes; external-bot adapter changes; any "deterministic Tier 0" labeling (ruling guard 1 — the licensed register is _grounded input + deterministic post-checks + explicit abstention_). The delta-4 `skipped-not-gated` parity-readout field rides this PR train as a separate small commit, outside this design.

### Data model deltas

1. **`VerdictArtifactSchema`** (new — `packages/core/src/artifacts/verdict.ts`), `VERDICT_ARTIFACT_SCHEMA_VERSION = '1.0.0'`, tolerant-within-major reader + migration registry, matching run/panel conventions.
   - `schemaVersion`; `createdAt` (excluded from content identity).
   - `diffScope` — **source-discriminated** (`explicit-range | staged | uncommitted | branch-vs-base`, the four `getDiffForReview` sources); `diffHash` always required, ref fields (`base`, `head`) required only where the source makes them meaningful. `getDiffForReview` is extended to return resolved scope metadata rather than reconstructing it later. **`diffHash` is computed over the exact masked `<git_diff>` segment delivered in the shared per-lane prompt** — assembled and DLP-masked ONCE pre-fan, identical bytes handed to every lane, INCLUDING the truncation marker when `MAX_DIFF_CHARS` truncation applies — reproducible from the persisted masked input, never binding bytes a lane did not see and never binding secret-bearing bytes (agy fold 5; codex rev-2 fold 4).
   - `lanes[]` — **discriminated union**, no impossible records (codex fold 2): `completed { laneId, resolvedBackend, runArtifactHash, verdictSummary }` · `abstained { laneId, resolvedBackend, runArtifactHash, reason }` (invoke happened, no usable structured verdict) · `failed { laneId, typedReason, resolvedBackend? }`. `superRefine` validates `attemptedLaneCount === lanes.length` and derives/validates `completedLaneCount` — counts are never mirrored on trust.
   - `panelArtifactHash` — present **iff** a #2104 panel was actually assembled (≥2 usable lanes); failed/abstained lanes are never handed to `assemblePanelArtifact`.
   - `postChecks[]` — reuses the persisted #2103 vocabulary (`ruleName`, `tier`, `verdict`, `message`); no renamed fields, no dropped diagnostics.
   - `findings[]` — normalized from the shared extractor (severity/confidence/file/line/message); panel diversity is a **top-level summary reference**, not mirrored per finding.
   - `round { index, priorVerdictHash?, lineageKey }`, `reviewedState: 'matched' | 'drifted'` (post-fan tree compare against the pre-fan content hash — a derived non-sentinel field; codex rev-2 fold 1), and `settled: boolean` (all derived — see lifecycle).
   - **No warm/cold discriminator — keys AND values** (strategy-codex G1): _consumers can identify which backend participated, never whether the producer was warm/cold/headless._ `laneId` is constrained to backend-derived vocabulary (validated shape, e.g. `lane-<n>:<provider>:<model>`), closing the value-smuggling channel the rev-2 structural key test missed. Enforced by the structural key test + a laneId-vocabulary refinement + schema JSDoc citing Prop 302.
2. **Config:** `review.lanes?: string[]` becomes a **declared key** in core `ReviewConfigSchema` — nonempty array of nonempty strings (`lanes: []` is a hard config error, present means ≥1; totem-codex finding 11) — with a single post-parse validator framed as **capability admission, not a provider allowlist** (strategy-codex G2): entries must parse as `provider:model`; the `shell` provider fails as an *unsupported adapter for fan lanes* (a support-limit error, never "structurally ineligible"); duplicates rejected on the normalized form. Absent ⇒ single lane resolved exactly as today (zero-regression default). **`--model` precedence pinned:** an explicit `--model` selects a one-lane invocation; it never joins or overrides an element of the configured fan. **Flag honesty on the fan path (finding 12):** `--raw` runs the legacy zero-LLM context dump — zero invokers, no artifacts (finding 1); `--out` writes the fan's human-readable report; `--suppress`, `--learn`, and `--auto-capture` are rejected loudly at startup when the fan is active (no silent acceptance) until they gain defined fan semantics.
3. **New on-disk container:** `.totem/artifacts/verdicts/<sha256>.json`, content-addressed, `wx` create-exclusive. EEXIST is **logical-identity dedup** (`createdAt` excluded from the address): on EEXIST, verify equality modulo `createdAt`; mismatch is a hard error (codex fold 8 / agy fold 4).
4. **Skill:** `review-loop` in `DISTRIBUTED_CLAUDE_SKILLS` + its `doctor --parity` mechanical-contract row + init distribution tests. The covariate line is **core-owned** (strategy-codex G4): `renderCovariateLine(verdict)` lives beside the schema so every caller — CLI print, headless, `/review-reply` — emits the identical signal; the skill is pure transport; the R3 pilot derives before-external ordering from timestamps, never from line presence. At settle the skill **holds the rendered line locally** (the CLI prints it; `/review-reply` re-derives it from the latest verdict artifact for the branch lineage when assembling the consolidated comment — a mechanism, not an instruction; totem-codex finding 14) — `local-lane: <verdictHash8> round=<n> settled=<true|false> lanes=<completed>/<attempted>` — and **never creates a PR comment itself**: the local loop runs before the external pass, and the round-disposition comment is ONE consolidated comment owned by the operator-invoked `/review-reply` workflow, which gains a one-line instruction to carry any pending `local-lane:` line into it (codex rev-2 fold 3 — single-comment ownership per bot-protocols; no autonomous GitHub mutation from `review-loop`). The line stays the stable grep-able pointer the pilot ledger joins on; the artifact stays canonical; **the line format is contract, versioned with the skill** (strategy change 3). Until a second warm agent passes the Prop 302 T2 falsifier, public support claims stay constrained to the verified agent(s) (codex, Tenet 19).
5. **New pure seams (codex folds 1/3):** `computeReviewedContentHash(...)` extracted from `writeReviewedContentHash`; **one shared review-output extractor** (XML-wrapped / fenced / bare JSON — the existing `extractStructuredVerdict` cascade) used by both the CLI path and a review-specific decidable post-check (the shipped generic `structuredOutputRule` is NOT wired for this — it bare-`JSON.parse`s and would mis-verdict valid Shield output); `handleVerdictResult` refactored so a pure function returns the lane result (extracted verdict, exemption-filtered findings, conformance) before any display/cache/throw side effect.

No reserved keys or sentinel values; separate fields throughout.

### State lifecycle

- **Two hash domains, explicitly separate (codex fold 1 — load-bearing):** `diffScope.diffHash` (masked review-payload identity) vs `.reviewed-content-hash` (extension-scoped tracked-source content hash that authorizes agent push). They bind different state; no code or test may claim equality. Today's shipped code recomputes the content hash **after** the LLM returns — the mid-run authorization race is real and this slice closes it: content hash computed **once, pre-fan**; compared post-fan; on drift → surface `worktree-drift`, write the verdict (bound to the reviewed diff), **do not stamp the cache**; on equal → stamp the **pre-fan** hash via the explicit writer. Never recompute inside the PASS side effect. Concurrency: no lock held across the fan; the final compare-and-stamp is a short critical section — each concurrent run binds to its own pre-fan hash and the stamp lands only if the tree still matches it (reconciles agy fold 4 without a minutes-long lock).
- **Verdict artifact:** persistent, immutable; a new round writes a new artifact. Assembled fully in memory → one `wx` write; no partial verdict can exist. A verdict is written only after every attempted lane reaches a terminal status.
- **Fan lanes:** strict runner wrapper over `runOrchestrator` — fan lanes force a fresh invoke (a response-cache hit emits no run artifact and therefore cannot produce a `completed` lane); missing artifact emission (`onEmitted` never fired) is a terminal lane failure, not WARN-and-continue; `runOrchestrator`'s global best-effort contract is untouched (14 other callers). "No retry" means no runner retry beyond the existing logged quota fallback; `resolvedBackend` records what actually ran.
- **Round chain:** lineage key is **composite over the resolved scope selector** (agy fold 3; codex rev-2 fold 2) — a domain-tagged hash over `{ repo/worktree identity, branch (or detached marker), source, mergeBase, base?, head? }`: explicit-range contributes its normalized base+head selectors; branch-vs-base its resolved base; staged/uncommitted the worktree identity (+ branch when present). Selector fields describe the _lineage_, never the changing diff bytes — legitimate fix rounds still chain. Implicit prior = latest verdict with the same lineage key; `--continues <hash>` is the explicit override; lineage divergence (branch rename, rebase moving the merge base) forks the chain to round 0 with a warning. Two branches sharing `base=main` can never cross-link, and neither can two different explicit ranges on one branch.
- **`settled` (pinned — the round-1 convergence finding):** a **current-round dryness predicate over artifact content only**, no cross-round input, no disposition input: `settled = (every attempted lane completed) AND (zero actionable findings after the existing exemption filter) AND (no decidable-tier post-check failure) AND (reviewedState === 'matched')`. Severity map pinned for R2: `INFO = cosmetic`, `WARN/CRITICAL = actionable`. The `reviewedState` clause (codex rev-2 fold 1): a dry fan over a tree that mutated mid-fan is NOT settled — the verdict does not cover the current tree, and the thin skill terminates on settled, so drift must fail the predicate, not just the cache stamp. The cross-round findings delta is **telemetry only**, never the termination predicate — a persistent CRITICAL can never settle by lane dropout (agy fold 1 satisfied structurally: a failed/abstained lane ⇒ fan incomplete ⇒ not settled), and a crafted lane output cannot flip it (strategy pin: pure function, option (a) — "dispositioned" language dropped; the exemption filter is the only removal mechanism, #2105 not implied).
- **Cache eligibility (distinct, weaker predicate — codex fold 4):** conjunction, never a vote: every attempted lane terminal AND usable (none `failed`/`abstained`) AND zero CRITICAL findings post-exemption AND no decidable post-check failure AND `reviewedState === 'matched'` (post-fan tree unchanged). Sensor-tier rows never affect it. A degraded fan (any lane failed) writes an honest verdict but **never stamps the cache** — the existing trap-ledgered `--override <reason>` remains the sole escape hatch (reconciles agy fold 2 without a new `--allow-degraded` flag). `settled` (no WARNs) is deliberately stricter than cache-eligible (no CRITICALs — today's PASS semantics): the drip class the runner absorbs is WARN-shaped.
- **Loop state:** entirely CLI-owned (gemini): round linkage, settle computation, and a max-rounds **advisory sensor line** (never a block) live in `totem review`; the skill is a thin driver.

### Failure modes

| Failure                                                            | Category  | Agent-facing surface                                                                                                                                                | Recovery                                                                   |
| ------------------------------------------------------------------ | --------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------- | -------------------------------------------------------------------------- |
| One lane fails (quota / network / key)                             | transient | loud warning; lane `failed` with typed reason; attempted ≠ completed in verdict; **no cache stamp**                                                                 | next round re-attempts; `--override` (trap-ledgered) is the audited escape |
| ALL lanes fail | transient | **honest verdict written** (all lanes terminal-failed, `settled=false`, never cache-eligible) + trailing hard error — "no verdict" is reserved for pre-attempt failures (config error, no diff); total failure is exactly when abstention provenance matters (strategy-codex G3) | fix backend, rerun |
| Lane output unextractable by the shared extractor                  | runtime   | lane `abstained` + reason; review-specific decidable post-check records it; no cache                                                                                | next round                                                                 |
| Working tree drifts mid-fan                                        | runtime   | `worktree-drift` surfaced; verdict written bound to pre-fan diffHash with `reviewedState='drifted'` and `settled=false` (even if otherwise dry); **no cache stamp** | next invocation = new round                                                |
| Response-cache hit on a fan lane                                   | —         | cannot occur (fresh invoke forced); artifact-emission failure ⇒ lane `failed`                                                                                       | n/a                                                                        |
| Concurrent invocations                                             | runtime   | each binds its own pre-fan hash; compare-and-stamp critical section; verdict EEXIST verified modulo `createdAt`                                                     | loser's stamp is skipped, its verdict stands                               |
| Verdict EEXIST, bytes differ modulo `createdAt`                    | permanent | hard error (identity violation)                                                                                                                                     | investigate — should be unreachable                                        |
| Prior verdict in chain missing/corrupt                             | permanent | warning; chain restarts at round 0                                                                                                                                  | none needed                                                                |
| Lineage divergence (rebase / branch change)                        | runtime   | warning; fork to round 0                                                                                                                                            | none needed                                                                |
| `review.lanes` invalid (unknown provider, shell, duplicate, empty) | init      | hard error from the config-boundary validator                                                                                                                       | fix config                                                                 |
| Single configured/available lane                                   | —         | verdict emitted; top-level degenerate-diversity sensor line (warnings-class, never a block)                                                                         | add lanes                                                                  |
| Round count exceeds advisory threshold                             | —         | max-rounds advisory sensor line                                                                                                                                     | human judgment                                                             |

### Invariants to lock in via tests (codex's eight gates adopted verbatim, plus panel additions)

1. Mid-fan tree mutation: verdict binds the pre-fan diff; no cache stamp authorizes changed content.
2. A response-cache/TTL hit cannot produce a `completed` lane without a genuine run artifact.
3. Valid XML-wrapped / fenced Shield verdicts pass the shared extractor; malformed output becomes `abstained` and cannot stamp the cache.
4. One failed + one passing lane: honest counts, verdict written, panel assembled only from usable lanes, no cache stamp.
5. A CRITICAL persisting across two rounds never sets `settled`; a genuinely dry current round does — including when a previously-failing lane completes.
6. Two branches sharing `base=main` cannot cross-link (composite lineage key).
7. Compatibility promise (precise, not "byte-for-byte" — codex rev-2 fold 5): with `review.lanes` absent, existing findings/display, exit, and cache semantics are unchanged and NO verdict artifact or `local-lane:` summary line is emitted — the runner contract requires `review.lanes` (a single entry suffices), and `review-loop` instructs configuring it when absent. With lanes configured, the verdict artifact and the additive `local-lane:` machine-readable line are the pinned new surface. `--model` precedence locked (one-lane invocation, never joins the fan).
8. Sensor-tier post-check failure never gates; decidable-tier can (ADR-109 / #2106 constraint 3).
9. Identical round inputs → identical verdict hash; `createdAt` never participates in identity; EEXIST-with-differing-bytes (modulo `createdAt`) hard-errors.
10. Structural schema test: no runner-lane discriminator field exists; JSDoc carries the deliberate-exclusion note citing Prop 302.
11. Lane union: `completed` requires `runArtifactHash`; `attemptedLaneCount === lanes.length`; `completedLaneCount` derived, not trusted.
12. `diffHash` is reproducible from the masked payload recorded in the artifact chain (hash symmetry).
13. A dry fan over a mid-fan-mutated tree yields `reviewedState='drifted'`, `settled=false`, and no cache stamp (codex rev-2 gate 1).
14. Two different explicit ranges on one branch + merge-base cannot cross-link (codex rev-2 gate 2).
15. A diff exceeding `MAX_DIFF_CHARS` with a secret beyond the truncation boundary: the stored `diffHash` recomputes exactly from the persisted masked input and never binds unseen bytes (codex rev-2 gate 4).
16. `review-loop` creates no PR comment; the covariate line reaches GitHub only inside the `/review-reply` consolidated round-disposition comment (codex rev-2 gate 3).
