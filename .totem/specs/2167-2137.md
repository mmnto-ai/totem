# Spec: distributed cohort-freeze read + freeze-aware staleness gate (#2167 + #2137)

Lane: strategy#584 emit/read half (gate-1, skynet sequencing). Sub-task 1 (emit) rides totem-strategy; sub-tasks 2–4 land here. Generated seed: `totem spec 2167 2137` (run 940ec4c9647e), corrected by hand where the #2137 issue body predated the 2026-06-12T0014Z ruling.

## Problem statement

The `rule-compilation` freeze is a cohort fact stored as a strategy-repo-local file; sibling repos cannot derive it (totem#2127 incident; lc#483/#497/#501 recurrence). Two halves ship here: (read) consumers sense cohort-scoped freezes from the installed `@mmnto/strategy-doctrine` snapshot and surface them in orient/doctor with provenance; (gate) `verify-manifest` becomes freeze-aware — **lesson-only staleness downgrades block → warn while `rule-compilation` is frozen** (the 0014Z ruling; push proceeds, zero compile invocation, zero artifact churn). The manifest-acknowledge shape in #2137's original body is DEAD — superseded by the gate-verdict change.

## Ruling constraints (strategy#584, 2026-06-12T0014Z)

- Lesson-only staleness (input-hash mismatch only) + active `rule-compilation` freeze ⟹ **WARN, exit 0**. Tenet 4 kept: sensed and reported, never silent.
- **Non-lesson drift still blocks**: output-hash mismatch (hand-touched `compiled-rules.json` / manifest) FAILs regardless of freeze state (Tenet 20 — regenerable cache stays protected).
- At unfreeze, accrued lesson staleness blocks again until a real compile settles it (the settle belongs to the spine-lift, strategy#516).
- Rejected by ruling: freeze-conditional staleness sensing (Tenet 13 — sensors report truth, actuators consult freeze); non-compile manifest re-stamp (hand-writing a regenerable cache).

## Architectural context

- **P288 §6.5**: freeze.json primitive — `{ _note, frozen[] }`, entries `{ subsystem, since, reason, tracking, do-not[] }`. Strategy's live file already carries `scope: "cohort"` on the rule-compilation entry (its `_note` documents the convention and points at strategy#584).
- **P292 §10.6**: the snapshot channel. `tools/build-strategy-doctrine.cjs` (strategy repo) is multi-artifact by design; the emit sub-task adds `{ canonical: '.totem/freeze.json', out: 'freeze.json' }` as a third `ARTIFACTS` entry — artifacts snapshot to the **package root under basename** (`node_modules/@mmnto/strategy-doctrine/freeze.json`), normalized, hash-bound in `strategy-doctrine.lock`.
- **Byte-copy invariant**: the §10.2 bind verifies `hash(canonical@last-published-sha) == content-hash`, so the emit NEVER filters — consumers filter to `scope: cohort` at read time. Strategy-local parked entries must not leak into sibling renders.
- **ADR-109**: repo-local freeze gate contract (fail-closed on corrupt local file) is UNTOUCHED. The distributed reader has its own, deliberately different failure contract (below).
- **Resolution pattern**: upward `node_modules` walk per `parity-detect.ts` (`resolveInstalledVersion` seam, GCA #2071 hoisting catch). Zero-network, never-throws, honest-absent.
- **Gate surfaces**: pre-push hook (`buildPrePushHook` → `$TOTEM_CMD verify-manifest`), CI (`.github/workflows/compile-manifest.yml:57`), manual — all three call the same command, so the verdict change ships once in the CLI with no hook re-install (the #2138 drift class is avoided by construction).
- **#2142 stopgap boundary**: compile's freeze no-op discharged the SAFETY half (no LLM rule-gen); this lane discharges the INVOCATION half (consumers stop invoking compile machinery at all). The stopgap stands.

- [ ] **Task 2: distributed cohort reader** (`packages/core/src/freeze.ts`)
  - `readCohortFreezes(cwd, packageName)`: upward-walk resolve `packageName`, read `<pkg-root>/freeze.json`, validate, filter `scope === 'cohort'`, return a STRUCTURED result `{ status: 'ok' | 'absent-package' | 'absent-file' | 'corrupt', entries, packageVersion?, warnings[] }`. The package name is caller-injected (gemini fold: core stays cohort-agnostic; the CLI supplies `COHORT_DOCTRINE_PACKAGE = '@mmnto/strategy-doctrine'`).
  - Failure contract (deliberately diverges from the local reader's fail-closed): absent-package / absent-file → honest-absent, distinguishable states; corrupt/invalid → `status: 'corrupt'` + warning strings IN the result — core never console-logs; surfaces render warnings deliberately (codex W1). Never throws, zero-network.
- [ ] **Task 3: effective-freeze union + provenance** (`packages/core/src/freeze.ts`)

Local entries (provenance `local`) ∪ cohort entries (provenance `cohort@<version>`). No dedup, no demotion — same-subsystem collisions render twice with distinct provenance. The local read keeps its fail-closed throw.
- [ ] **Task 4: orient + doctor surfacing** (`packages/cli/src/commands/orient.ts`, `doctor.ts`)
  - Orient PARKED block unions via `readEffectiveFreezes`, tagging `[cohort @ strategy-doctrine <v>]`; SessionStart payload renderer inherits. Doctor gains a sensor-only freeze line (never gated, never strict-promoted). Both surfaces render `result.warnings` + non-`ok` channel statuses distinctly in human AND structured output (codex W1: no stderr-crumb diagnosis).
- [ ] **Task 5: freeze-aware verify-manifest verdict (#2137, the 0014Z ruled shape)** (`packages/cli/src/commands/verify-manifest.ts`)

## Implementation Design

### Scope

Ships the totem-side read half (freeze scope/id schema, distributed cohort-freeze reader, orient/doctor surfacing) and the freeze-aware verify-manifest verdict, closing #2167 + #2137. Explicitly NOT: the strategy-side emit (separate PR), any ADR-109 gate-engine change, any compile-path change (#2142 stands), lock content-hash verification (#2107), unfreeze-settle automation (strategy#516), and no new hook text (verdict ships inside the existing `verify-manifest` call).

### Data model deltas

| Delta                                                                      | Holds              | Written by             | Read by                       | Invariants                                                                                                                     |
| -------------------------------------------------------------------------- | ------------------ | ---------------------- | ----------------------------- | ------------------------------------------------------------------------------------------------------------------------------ |
| `FreezeEntrySchema.scope: 'local' \| 'cohort'` (optional, default `local`) | distribution class | freeze-file authors    | reader filter, renderers      | additive; absent = `local` (legacy-safe — today's `z.object` already strips it)                                                |
| `FreezeEntrySchema.id` (optional kebab slug)                               | stable machine key | freeze-file authors    | gate consumers                | strict-at-emit (strategy build validates cohort ⇒ id), lenient-at-read (cohort-without-id renders + warns, never gate-matches) |
| `ActiveFreeze { entry, provenance: 'local' \| 'cohort', sourceVersion? }`  | union row          | `readEffectiveFreezes` | orient/doctor/verify-manifest | unbounded by file size only; never persisted                                                                                   |
| `EffectiveFreezeResult { entries, localStatus, cohortStatus, cohortPackageVersion?, warnings[] }` (codex W1) | union + per-source status | `readEffectiveFreezes` | ALL surfaces (human, structured, gate) | statuses survive into every consumer; warnings returned in the result, never console-logged from core |
| `RULE_COMPILATION_FREEZE_ID = 'rule-compilation'` (core constant)          | gate match key     | n/a (constant)         | verify-manifest               | the ONLY freeze id the gate consults in this slice                                                                             |

No reserved keys, no sentinel values, no new persisted artifacts on the totem side.

### State lifecycle

All state is per-invocation: both readers run fresh per command (resolver purity matches the #2141 precedent — no module-level caches), the union is computed and discarded, and verify-manifest's freeze context lives only for the verdict. The only cross-lifecycle artifact is the npm snapshot itself (per-install lifetime, owned by pnpm; staleness sensed by the existing parity-currency contract, not by this code).

### Failure modes

| Failure                                                     | Category | Agent-facing surface                                                                                 | Recovery                           |
| ----------------------------------------------------------- | -------- | ---------------------------------------------------------------------------------------------------- | ---------------------------------- |
| local freeze.json corrupt                                   | runtime  | hard error (UNCHANGED — ADR-109 fail-closed)                                                         | fix the file                       |
| snapshot package not installed                              | runtime  | doctor/orient: honest-absent note; gate: blocks as today                                             | install/bump optionalDep           |
| package installed, no freeze.json inside (pre-emit version) | runtime  | doctor: distinct honest-absent note ("snapshot predates freeze distribution"); gate: blocks as today | pin-bump at next doctrine cut      |
| snapshot freeze.json corrupt / schema-invalid               | runtime  | LOUD WARN in every reader; entries not visible; gate blocks (conservative)                           | republish; pin-bump                |
| cohort entry without `id`                                   | runtime  | renders in orient/doctor + WARN naming the entry ("no stable id — machine consumers cannot bind")    | strategy adds id at next cut       |
| freeze active + input-hash-only mismatch                    | runtime  | WARN, exit 0 — names freeze, tracking, post-spine resume                                             | real compile at spine-lift         |
| freeze active + output-hash mismatch (± input)              | runtime  | FAIL (unchanged)                                                                                     | restore compiled-rules.json        |
| no freeze visible + any mismatch                            | runtime  | FAIL byte-identical to 1.61.0                                                                        | unchanged                          |
| unexpected throw inside the freeze consult                  | runtime  | loud warn + conservative FAIL path (no-freeze-visible)                                               | report; gate never silently passes |

No silent-degradation rows: every absence/corruption state is rendered distinctly (Tenet 4); the two honest-absent package states are deliberately distinguishable so "channel not adopted" ≠ "channel adopted, snapshot too old".

### Invariants to lock in via tests

1. Legacy freeze entries (no `scope`) parse as `local`; the strategy live-entry shape round-trips with `scope`/`id` preserved.
2. A `scope: local` entry inside the distributed snapshot NEVER surfaces in a consumer repo (leak filter).
3. Corrupt distributed freeze: no throw, warn emitted, zero entries visible, gate verdict stays blocking.
4. Union never demotes: same-subsystem local + cohort entries both render, distinct provenance.
5. Gate verdict matrix: (freeze ∧ input-only) ⟹ exit 0 + warn; (output mismatch) ⟹ throw regardless of freeze; (no freeze ∧ input-only) ⟹ throw with the 1.61.0 message byte-identical.
6. Gate matches by `id` only: reworded `subsystem` with intact `id` still matches; id-less cohort entry never matches.
7. Both readers are zero-network and write nothing to disk.
8. Resolver purity: a snapshot installed between two calls is visible to the second (no caching).
9. Per-source statuses survive to every surface: corrupt distributed snapshot ⟹ doctor WARN + orient visible warning + verify-manifest warns-while-blocking; absent-package vs absent-file render distinctly; genuinely-none is distinct from both (codex W1).
10. The gate's match constant is imported from core — a grep for the `rule-compilation` literal in `packages/cli` finds only the import site (codex W3).
11. Core emits no console output for channel states; every warning rides a structured result (codex W1).

### Open questions — resolved above

1. **Gate match key.**
   - **Options:** (a) optional stable `id` slug, strict-at-emit / lenient-at-read (spec'd above); (b) exact full `subsystem` string constant (ADR-109 V1 style) — silently unbinds if strategy rewords the prose; (c) prefix/token match on `subsystem` — the mis-bucket class the #2141 filename-token review just killed.
   - **Recommendation:** (a). The conformant-degraded discipline mirrors the ADR-098 filename-token settle; ADR-109's exact-match contract stays untouched because the gate-engine is out of scope.
2. **Unfreeze propagation window.** When strategy lifts the freeze, consumers keep warning (not blocking) until their snapshot pin bumps.
   - **Options:** accept the window (pin currency is already a sensed parity contract; zero-network preserved) vs. any network check (violates the channel's core invariant).
   - **Recommendation:** accept; the window is bounded by the cohort-floor pin-bump cadence and the gate restores blocking automatically at the bump. Worth an explicit operator nod since it briefly extends warn-mode past the lift.
3. **Provenance breadth for the gate.** Should a repo-LOCAL `rule-compilation` freeze (id-matched) also downgrade the gate, or cohort-only?
   - **Options:** any-provenance (freeze is freeze; provenance is display) vs. cohort-only (narrowest reading of the ruling).
   - **Recommendation:** any-provenance — a repo that deliberately freezes its own compile path wants the same non-invocation behavior; the ruling's cohort framing is about distribution, not about local freezes being weaker.
