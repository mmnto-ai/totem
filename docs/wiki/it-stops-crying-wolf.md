# It Stops Crying Wolf (The Self-Healing Loop)

The graveyard of developer tooling is filled with Static Analysis (SAST) and linting tools that failed for one reason: **Alert Fatigue.**

If a tool throws 50 warnings about a "security vulnerability" that the developer knows is just a test fixture, the developer stops trusting the tool. They learn to ignore it, add blanket suppression comments, or use `--no-verify` to bypass the hook entirely.

Traditional linters are static. They cry wolf until a senior engineer spends a weekend manually tuning the configuration file.

**Totem is a Self-Healing System.** It observes how developers interact with its rules and automatically tunes itself to stop crying wolf.

## The Trap Ledger

When `totem lint` or `totem review` blocks a developer, they are not permanently stuck. They can use an escape hatch:

```typescript
// totem-context: This is a synthetic test fixture, not a real AWS key.
const key = 'AKIAIOSFODNN7EXAMPLE';
```

Or via the CLI:

```bash
totem review --override "False positive: onWarn is in scope at line 273."
```

Instead of just bypassing the check, Totem intercepts this action and logs it to the **Trap Ledger** (`.totem/ledger/events.ndjson`). This turns a frustrated developer bypass into highly structured training data.

## The Rule Nursery

Rules generated by AI (via `totem lesson compile`) are inherently experimental. When a new rule is born, it enters the **Rule Nursery**.

Periodically, you run the doctor command:

```bash
totem doctor --pr
```

Totem analyzes the Trap Ledger. If it sees that Rule #42 (e.g., "No hardcoded vendor secrets") has fired 100 times, but developers have used `// totem-context:` to suppress it 95 times in `**/*.test.ts` files, the system realizes the rule is overly broad.

## The Auto-Downgrade

Instead of requiring human intervention, `totem doctor` autonomously opens a Pull Request:

1.  It automatically downgrades the noisy rule's severity from `error` (blocking) to `warning` (advisory).
2.  It suggests a more precise file scope (e.g., explicitly excluding `!**/*.test.ts`).
3.  It updates the source markdown lesson so the mistake isn't repeated.

## The Result

Totem's enforcement layer gets quieter and more accurate over time. It only blocks builds for rules that the team actually respects.

It stops crying wolf, so when it _does_ block a push, developers know it's a real problem.
