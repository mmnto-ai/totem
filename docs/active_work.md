### Next Issue (User Story & Scope)

**#168 — UX: Interactive multi-select pruning for `totem learn`**

- **User Story:** As a developer running the `totem learn` loop, I want an interactive checklist to prune hallucinated or low-value lessons generated by the AI before they are saved to the vector index, so I don't have to choose between rejecting the entire batch or manually editing the `lessons.md` file later.
- **Scope Boundaries:** Adopt `@clack/prompts` to replace the simple `(Y/n)` prompt in `packages/cli/src/commands/learn.ts` with an interactive multi-select menu. Parse the LLM's markdown response into individual lesson objects so they can be rendered as selectable items. Ensure the `--raw` non-interactive mode still functions safely.
- **Why Next:** As AI agents increasingly drive the `learn` command autonomously, the human-in-the-loop approval step must become more granular to maintain high-quality memory without slowing down momentum.
